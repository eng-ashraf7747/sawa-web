# \## الحادثة #1 — افتراض تطابق اسم Collection في الكود المصدري مع البيانات الفعلية المنشورة

# 

# \*\*التاريخ:\*\* يوليو 2026

# 

# \*\*الخطأ:\*\*

# لاحظنا إن شاشة "سجل نقاطي" مش بتعرض مصادر النقاط. الكود المصدري لدالة `onReferralRequest`/`onUserRegistered` (في `functions/src/config/referralConfig.ts`) بيقول إن اسم الـ Collection هو `"pointsLedger"` وإن حقل الحالة اسمه `type: "signup\_bonus"`. \*\*بنيت على الافتراض ده وعدّلت كود الواجهة (`usePoints.ts`) ليطابقه — من غير ما أتحقق من البيانات الفعلية المخزّنة في Firestore أولاً.\*\*

# 

# \*\*النتيجة:\*\* التعديل خلّى الواجهة تقرأ من Collection تاني تمامًا (`pointsLedger` بالحروف المتلاصقة) غير اللي فيه البيانات الحقيقية فعلاً (`points\_ledger` بالشرطة التحتية، بحقل `type: "earned"` + `source: "registration"`) — يعني \*\*الكود المصدري في الريبو كان متأخر عن آخر نشر فعلي لدوال Firebase\*\*، وده فرق ما كنش متوقع.

# 

# \*\*السبب الجذري:\*\*

# Firebase Cloud Functions ليها دورة نشر مستقلة تمامًا عن Next.js/Vercel (`firebase deploy --only functions`) — ممكن الكود في الريبو يتغيّر (يتكتب أو حتى يتعمله commit) من غير ما حد ينشره فعليًا على Firebase. يعني \*\*الكود المصدري لدوال Firebase مش دليل موثوق 100% على شكل البيانات الفعلي المُنتَج فعليًا في قاعدة البيانات\*\*، عكس كود Next.js اللي بمجرد ما يتعمله push وBuild ينجح، بيبقى هو نفسه اللي شغال على الإنتاج (طالما Vercel نشره بنجاح).

# 

# \## القاعدة الجديدة (إلزامية من الآن)

# 

# \*\*قبل أي تعديل على منطق قراءة/كتابة بيانات مرتبط بـ Cloud Functions (اسم Collection، شكل الحقول، القيم المحتملة)، لازم نتحقق من البيانات الفعلية في Firestore Console مباشرة أولًا — مش من الكود المصدري لدوال Firebase وحده، مهما بدا واضحًا ومباشرًا.\*\*

# 

# بالتحديد:

# 1\. لو الشك حوالين اسم Collection أو شكل حقل، \*\*نفتح Firestore Console ونشوف مستند حقيقي بعينه\*\* قبل أي تعديل كود.

# 2\. أي تناقض بين الكود المصدري لدوال Firebase وبين البيانات الفعلية = إشارة إن الدوال المنشورة فعليًا (Deployed) متأخرة عن الريبو، مش إن البيانات "غلط" — والتعامل مع الحالة دي بيكون بمطابقة الكود القارئ للبيانات الحقيقية أولًا (إصلاح فوري وآمن)، وتسجيل مهمة منفصلة لاحقًا لمزامنة/نشر كود الدوال الصحيح.

# 3\. \*\*هذا التحقق إلزامي بغض النظر عن مدى وضوح الكود المصدري أو الثقة فيه\*\* — الثقة في القراءة مش بديل عن التحقق الفعلي من مصدر الحقيقة الوحيد: قاعدة البيانات نفسها.
